# Program 1 Architecture Review Track Phase I0 - Scope and Boundaries

Status: documentation-only, synthetic-only, non-production, non-runtime, pre-implementation, conceptual only.

Phase I0 opens Program 1 Architecture Review Track Phase I. Phase I follows Phases A through H. It is not Phase Z+1 and is not a return to the Phase V-Z governance/prototype sequence.

Program 1 remains in pre-implementation hold.

## In scope

- Conceptual integration boundary.
- Conceptual connector and API prohibition.
- Conceptual database and EHR/EMR boundary.
- Conceptual patient portal, appointment, messaging, and external system boundary.
- External system dependency and control matrices.

## Out of scope

Runtime code, tests, helpers, scripts, imports, services, database migrations, API endpoints, UI flows, schedulers, task runners, integrations, connectors, API clients, database clients, EHR/EMR connectors, patient portal connectors, appointment system connectors, messaging system connectors, external system adapters, data connectors, real-data ingestion, PHI/PII processing, environment configuration, secrets, credentials, read-only runtime access, database queries, EHR/EMR access, patient messaging, appointment mutation, workflow enforcement, Task engine, Outcome Evidence, clinical write workflows, production-readiness, and go-live are out of scope.

## Non-approval

Phase I0 does not approve integration, connectors, APIs, database access, EHR/EMR access, patient portal access, appointment system access, messaging system access, external system access, runtime data exchange, production access, real-data access, PHI/PII processing, clinical deployment, go-live, or approval/clearance/override runtime capability.
