# Program 1 Phase J2 - Demo Data Boundary Synthetic Data Rules

## Demo Data Only

Demo data must be fake, synthetic and visibly safe for demonstration.

## Forbidden Real Data

- real patient names
- real DOB/OIB/MBO
- real email, phone or address
- real clinical documents
- real lab/imaging reports
- real patient messages

## Synthetic Convention

- use names such as `Demo Pacijent 001` or clearly fictional examples
- use impossible or labeled demo identifiers
- label clinical documents as demo/synthetic
- use demo timestamps and non-real source references
- avoid combinations that could identify a person

## Red Flags

Stop the demo if a user starts entering real identifiers, real clinical text, real contact data or any unapproved document.
