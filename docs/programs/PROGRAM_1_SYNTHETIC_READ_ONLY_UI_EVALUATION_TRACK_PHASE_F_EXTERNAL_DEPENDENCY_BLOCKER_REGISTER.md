## Remaining External Dependencies

1. A real adult clinician voluntarily agrees to participate.
2. Eligibility is confirmed without collecting unnecessary personal data.
3. The participant receives and accepts the synthetic-only consent script.
4. A moderator and stop authority are named.
5. A local machine and exact candidate SHA are assigned.
6. A session time is separately scheduled.
7. The session is performed using no real patient or practice information.
8. Controlled observations are reviewed by the named evidence and safety reviewers.

These dependencies cannot be completed by repository implementation or AI inference.

## Genuine Blocker

Program 1 cannot truthfully advance to an `external session complete`, `clinician evaluated`, or `clinical usability validated` state without the real participant, consent, observed session, and reviewer evidence.

The blocker is external human participation and authorization, not missing repository functionality.

## Forbidden Substitutions

- simulated clinician answers
- AI-generated participant quotes
- fabricated consent
- self-scored session presented as external evidence
- engineering test results presented as clinical validation
- synthetic runner state presented as a session record
