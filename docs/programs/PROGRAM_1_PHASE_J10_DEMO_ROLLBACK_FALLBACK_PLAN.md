# Program 1 Phase J10 - Demo Rollback Fallback Plan

## Stop Criteria

- real data appears
- permissions fail in a way that could confuse safety
- backend or database is unstable
- timeline/finding/open-question display cannot be explained safely
- stakeholder asks to use production or real data

## Fallback Options

- switch to release notes
- show architecture/governance docs
- explain API contracts
- use prepared screenshots if available
- schedule technical follow-up instead of live debugging

## Incident Reporting

Record the issue, environment, command/output where safe, and whether safety messaging was affected.
