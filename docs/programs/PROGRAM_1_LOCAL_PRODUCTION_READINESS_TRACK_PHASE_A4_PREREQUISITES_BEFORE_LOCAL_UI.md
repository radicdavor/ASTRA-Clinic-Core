# Program 1 Local Production Readiness Track Phase A4 - Prerequisites Before Local UI

Status: prerequisite list. No-UI Hold remains active.

Before any local UI may be considered, require:

- explicit authorization for UI Review Track
- UI purpose statement
- screen inventory
- no-patient-data UI boundary
- no-PHI/PII UI boundary
- no-persistence default
- no-network default
- no-patient-facing surface
- no clinical-action controls
- no send/schedule/approve/override buttons
- local-only execution plan
- UX safety labeling plan
- review of whether terminal remains sufficient

Phase A does not authorize UI code.

No web UI, browser UI, local web app, server/API, or hosted preview is authorized.
