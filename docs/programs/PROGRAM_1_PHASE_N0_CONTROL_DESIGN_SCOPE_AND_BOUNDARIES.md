# Program 1 Phase N0 - Control Design Scope and Boundaries

## Definitions

Control design:
Documentation of the intended governance control, owner, evidence, decision boundary, and closure gate.

Control implementation:
Actual software, operational, or procedural enforcement of the control.

Control validation:
Evidence that the implemented control works under expected and negative conditions.

Control approval:
Formal acceptance by required owners.

Production authorization:
A separate future decision that may only be considered after required controls are implemented, validated, and approved.

## Phase N Boundary

Phase N performs control design only.

Phase N does not perform implementation, validation, approval, or production authorization.

Phase N does not create production controls, runtime enforcement, clinical write workflows, approval/clearance/override logic, patient messaging, appointment mutation, Task engine, or Outcome Evidence.
