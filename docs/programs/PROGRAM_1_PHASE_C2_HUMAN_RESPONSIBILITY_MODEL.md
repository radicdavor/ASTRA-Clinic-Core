# Program 1 Phase C2 - Human Responsibility Model

Status: responsibility boundary model

## Purpose

This document defines who is responsible for interpreting Clinical Readiness signals.

## Physician Responsibility

The physician remains responsible for clinical decisions.

ASTRA may organize and display evidence, but it must not decide.

## Nurse And Reception Limits

Nurse and reception users may help collect information and identify missing operational inputs.

They must not be presented as clearing clinical readiness unless future governance explicitly allows a narrow nonclinical workflow.

## AI And API Key Limits

AI and API keys may not approve, clear, override or enforce readiness.

AI/API output must remain suggestion or data access within strict permissions.

## System Responsibility Limits

The system may:

- show advisory signals
- explain missing inputs
- point to source evidence
- record audit history

The system may never:

- make clinical decisions
- approve procedures
- clear readiness
- change appointment status
- create patient-facing messages

## Audit Responsibility

Audit records what happened.

Audit does not prove clinical outcome or readiness.

## Human-Mediated Future Enforcement

Any future enforcement must be human-mediated, reasoned, audited and separately approved.
