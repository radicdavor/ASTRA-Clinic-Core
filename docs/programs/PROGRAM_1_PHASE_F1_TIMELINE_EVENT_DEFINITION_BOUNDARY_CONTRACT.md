# Program 1 Phase F1 - Timeline Event Definition Boundary Contract

Status: documented

## Timeline Event

A timeline event is a source-linked display record derived from an existing object or contract. It describes that something relevant exists or changed in the clinical knowledge layer.

## Distinctions

- Source document: source material, not the event itself.
- Finding: source-linked clinical information.
- Open question: unresolved source-linked clinical question.
- Extraction candidate: passive non-official candidate.
- Review: human review context, not a decision.
- Readiness snapshot: advisory snapshot, not workflow enforcement.
- Acknowledgment: human reviewed a signal/context, not approval.
- Audit event: access/security evidence, not clinical outcome evidence.
- Physician decision and recommendation: separate future concepts.

## Explicit No-Go

Timeline event is not clinical decision, Task, Outcome Evidence, patient message, automatic diagnosis, automatic treatment or appointment status mutation.
