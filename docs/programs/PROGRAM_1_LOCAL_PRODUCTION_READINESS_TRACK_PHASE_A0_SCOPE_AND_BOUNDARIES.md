Status: documentation-only. Local-only boundary definition. Not deployment. Not go-live.

Allowed:

- local production-readiness documentation
- local-only boundary definition
- first local deployment candidate definition
- prohibited local production paths
- prerequisites before local UI
- prerequisites before real-data read-only
- prerequisites before clinical workflow behavior
- local operational risk register
- No-UI Hold continuity

- Python runtime code
- new CLI command
- web UI, browser UI, or local UI prototype
- server or API endpoint
- network or database access
- integrations
- persistence or export
- real-data ingestion or PHI/PII handling
- patient messaging or appointment mutation
- workflow enforcement
- clinical writeback or clinical task creation
- autonomous diagnosis, treatment, triage, or patient instruction
- production-readiness claim for clinical use
- go-live authorization
- approval/clearance/override capability
- cloud deployment or deployment automation

Current decision: documentation-only local production-readiness boundary definition.
