# Phase G — Canonical documentation consolidation

Program 2 now has three short, canonical current-state sources:

- [`docs/CURRENT_PRODUCT_STATE.md`](../../CURRENT_PRODUCT_STATE.md) — user-visible workflows, roles, routes and current disposition;
- [`docs/CURRENT_ARCHITECTURE.md`](../../CURRENT_ARCHITECTURE.md) — runtime components, data flow, source-of-truth hierarchy and provider boundaries;
- [`docs/CURRENT_OPERATIONAL_LIMITATIONS.md`](../../CURRENT_OPERATIONAL_LIMITATIONS.md) — explicit technical, operational, privacy, accessibility and evidence limits.

`README.md` links to these documents near its beginning. Program phase, roadmap and closure files remain historical evidence and carry a notice that they are not the current product-state authority. They were not deleted or rewritten as current approval.

The canonical posture is intentionally narrow: local/controlled synthetic use only, no live integrations, no real data, no production deployment and no go-live authorization. This consolidation does not replace the architecture Bible or migration history and does not certify clinical safety, privacy compliance or operational readiness.
