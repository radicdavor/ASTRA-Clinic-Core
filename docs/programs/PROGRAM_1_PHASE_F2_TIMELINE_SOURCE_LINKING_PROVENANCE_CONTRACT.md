# Program 1 Phase F2 - Timeline Source-Linking Provenance Contract

Status: documented

## Required Provenance

Timeline events must include source object type, source object id/reference, patient id, source document reference when available, event timestamp, display timestamp, created timestamp, provenance label and limitations.

## Traceability Rules

Timeline display must let a user understand where an event came from. Unreviewed sources must remain clearly labeled as unreviewed. Confidence and limitations should be displayed when applicable.

## Safety Boundary

Traceability does not make the event official clinical truth. Source-linked display remains context for human interpretation.
