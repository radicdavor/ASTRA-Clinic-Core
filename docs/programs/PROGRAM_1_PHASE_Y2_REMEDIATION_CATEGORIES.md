# Program 1 Phase Y2 - Remediation Categories

Phase Y uses these remediation categories to avoid accidental implementation.

| Category | Meaning | Current implementation permission |
| --- | --- | --- |
| documentation-only clarification | Clarify wording, links, scope, or blocker status. | allowed |
| governance review required | Future owner review is required before change. | docs only now |
| clinical safety review required | Clinical safety review must precede future consideration. | docs only now |
| privacy/legal review required | Privacy/legal review must precede future consideration. | docs only now |
| security architecture review required | Security design review must precede future consideration. | docs only now |
| technical design required | Technical design is needed before any implementation. | docs only now |
| runtime implementation deferred | Runtime work is intentionally postponed. | not allowed now |
| prohibited until separately approved | Capability remains no-go until explicit future approval. | not allowed now |

No category in Phase Y authorizes runtime implementation, production-readiness, real-data use, PHI/PII processing, clinical deployment, go-live, or approval/clearance/override capability.
