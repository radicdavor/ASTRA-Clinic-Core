# Program 1 Phase F13 - Timeline Workspace Contract

Status: documented, no UI

## Future Surface

The future read-only timeline may appear in Patient Workspace near documents, findings and open questions. It must remain source-linked and read-only.

## States

Future UI must define loading, empty, error and permission-denied states. Empty state must not imply no clinical concerns.

## Display

Events may be grouped by date, source document or source type. Every displayed event must show provenance and limitations.

## No Actions

No review, approve, clear, resolve, Task, Outcome Evidence or patient-message controls are allowed in this phase.
