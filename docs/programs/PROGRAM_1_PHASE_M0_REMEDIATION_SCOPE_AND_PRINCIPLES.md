# Program 1 Phase M0 - Remediation Scope and Principles

## Principles

- safety before feature expansion
- governance before automation
- validation before deployment
- human clinical responsibility before clinical use
- real-data governance before real patient data
- auditability before operational use
- rollback before release
- monitoring before live exposure
- least privilege before access expansion
- demo readiness is not production readiness

## Prohibited Interpretations

- A remediation plan is not an approval.
- A planned control is not an implemented control.
- An implemented control is not a validated control.
- A validated demo control is not necessarily a production control.

## Scope Boundary

Phase M may define workstreams, owners, sequencing and evidence requirements. It must not implement controls, close gaps, grant production approval, grant real-data approval or authorize clinical write workflows.
