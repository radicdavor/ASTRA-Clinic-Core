# Issue Labels

Recommended labels for closed pilot triage:

- `pilot:P0`
- `pilot:P1`
- `pilot:P2`
- `pilot:P3`
- `area:frontend`
- `area:backend`
- `area:inventory`
- `area:billing`
- `area:appointments`
- `area:audit`
- `area:security`
- `area:docs`
- `real-data-blocker`

Severity labels:

- `pilot:P0`: data corruption, security issue, or demo cannot proceed.
- `pilot:P1`: core workflow blocked.
- `pilot:P2`: confusing but workaround exists.
- `pilot:P3`: cosmetic or minor wording issue.

Apply `real-data-blocker` to any finding that must be resolved before real patient data can be considered.
