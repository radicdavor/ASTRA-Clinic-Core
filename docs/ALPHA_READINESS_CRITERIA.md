# Alpha Readiness Criteria

ASTRA Clinic Core may move from pilot planning into alpha planning only when all criteria below are met.

- No open P0/P1 pilot issues.
- At least one completed human pilot dry-run exists in `docs/pilot_sessions/`.
- Release notes are updated from the dry-run.
- Audit trail was accepted in pilot.
- Material/inventory workflow was accepted in pilot.
- Invoice/payment demo workflow was accepted in pilot.
- Real-data checklist still blocks real data unless formally reviewed and accepted.
- No real fiscalization claim is made while fiscalization mode is noop/stub.
- Demo banner and public config warning are visible in demo mode.
- P2/P3 issues are triaged and intentionally accepted or scheduled.

Alpha planning does not mean real patient data is allowed. Real-data readiness requires a separate formal review.
